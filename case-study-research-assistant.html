CASE STUDY 01

Research Assistant Usage API

Structured ingestion pipeline for operational field data submitted via PDF forms.

The Problem

Researchers submitted assistant usage records via Microsoft Forms. The submissions were converted to PDF and required manual extraction into spreadsheets — introducing inefficiencies and data inconsistencies.

System Architecture

FORM SUBMISSION
↓
PDF GENERATION
↓
POWER AUTOMATE TRIGGER
↓
GITHUB API (PDF PARSER)
↓
STRUCTURED DATA RETURN
↓
SPREADSHEET AMENDMENT

My Role

Collaborated on API structure and implemented the parsing logic responsible for extracting structured data from submitted PDFs. Designed validation patterns and ensured repeatable automation across the workflow.

Technical Stack

Python · REST API · PDF Parsing · Power Automate · Excel Automation

Outcome

Reduced manual data handling, improved operational accuracy, and established a repeatable ingestion architecture for assistant usage records.
